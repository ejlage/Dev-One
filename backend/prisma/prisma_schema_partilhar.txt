Como partilhar a Base de Dados (forma correta)

👉 Nunca partilhas a base de dados diretamente
👉 Partilhas a estrutura + forma de gerar

🔥 MÉTODO PROFISSIONAL (Prisma)
✔️ O que tu vais partilhar

No GitHub:

backend/
├── prisma/
│   └── schema.prisma   ✅
👨‍💻 O que os teus colegas fazem
🔹 1. Clonar projeto
git clone https://github.com/ejlage/Dev-One.git
cd Dev-One
🔹 2. Instalar dependências
npm install
🔹 3. Criar .env

Eles criam:

DATABASE_URL="postgresql://postgres:password@localhost:5432/entartes"
🔹 4. Criar base de dados

No psql:

CREATE DATABASE entartes;
🔹 5. Criar estrutura da BD (AUTOMÁTICO)

👉 Aqui está a magia do Prisma:

npx prisma db push
💥 Resultado

👉 Prisma cria automaticamente:

tabelas
relações
constraints

✔️ Igual ao teu .ddl

🧠 Alternativa (se quiseres usar o teu .ddl)

Também podes partilhar:

backend/schema.ddl

E eles fazem:

\i schema.ddl
⚠️ Mas melhor usar Prisma

👉 Porque:

versionado
mais limpo
mais moderno
💡 Dica TOP (mesmo importante)

Adiciona um ficheiro:

notepad README.md
